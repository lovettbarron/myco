# Phase 3 -- UI Design Contract

> Visual and interaction contract for the Webview Caps phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> 
> This phase adds three GPU-rendered elements (markdown viewer, file sidebar, focus desaturation) and one webview element (TLDraw canvas). The design contract covers both the Rust/wgpu rendering tokens and the minimal CSS contract for the TLDraw webview wrapper.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Rust + wgpu native rendering) |
| Preset | not applicable |
| Component library | none (custom Rust renderers: QuadRenderer, TextEngine via glyphon/cosmic-text) |
| Icon library | none (text-based labels, e.g. "X" for close; no icon font) |
| Font | System sans-serif (via cosmic-text FontSystem::new()) for UI; system monospace for code blocks |

**Source:** Existing codebase (src/renderer/text_renderer.rs, src/theme.rs). No web component library -- all rendering is GPU-native via wgpu.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline padding, icon gaps, sidebar item indent per depth level |
| sm | 8px | Compact element spacing, sidebar item vertical padding, markdown list indent |
| md | 16px | Default element spacing, markdown paragraph gap, sidebar horizontal padding |
| lg | 24px | Section padding, markdown heading top margin, code block padding |
| xl | 32px | Layout gaps, markdown H1 top margin |
| 2xl | 48px | Major section breaks (not used in Phase 3) |
| 3xl | 64px | Page-level spacing (not used in Phase 3) |

Exceptions:
- TITLE_BAR_HEIGHT = 38px (established Phase 1, not on scale -- platform-level constant)
- PANEL_TITLE_HEIGHT = 28px (established Phase 1, not on scale -- panel chrome constant)
- DIVIDER_VISUAL_WIDTH = 1px (established Phase 1 -- subpixel element)
- Close/fullscreen buttons = 16x16px at 6px inset from panel top edge (established Phase 1)
- SIDEBAR_WIDTH = 240px (new -- fixed sidebar width, outside 8-point scale due to content-driven sizing)
- Sidebar item height = 28px (matches PANEL_TITLE_HEIGHT for visual consistency)

### Accent Reserved For

The accent color (divider_hover: #666673) is reserved for:
1. Focused panel top indicator bar (2px, existing)
2. Hovered dividers (existing)
3. Sidebar selected item left edge indicator (2px vertical bar)
4. "New Canvas" button text highlight in sidebar

No destructive color needed in Phase 3. There are no delete/destroy operations in this phase (canvas auto-saves; markdown is read-only; panels close via existing Phase 1 mechanism).

### TLDraw Webview CSS Colors

The TLDraw webview wrapper uses its own React-based styling. Apply these CSS overrides to the wrapper HTML to match the Myco dark theme:

```css
html, body {
    margin: 0;
    padding: 0;
    overflow: hidden;
    height: 100%;
    background: #1E1E24; /* terminal background -- blends with panel */
}
.tldraw-container {
    position: fixed;
    inset: 0;
}
/* D-16: Desaturation for unfocused canvas panels */
body.unfocused {
    filter: saturate(0.3) brightness(0.7);
    transition: filter 150ms ease;
}
body.focused {
    filter: none;
    transition: filter 150ms ease;
}
```

TLDraw ships its own dark mode. Use `<Tldraw inferDarkMode />` to auto-detect, or force dark mode via `<Tldraw forceMobile={false} />` with CSS `prefers-color-scheme: dark` on the HTML element. Do NOT override TLDraw's internal component colors -- only the container background and desaturation filter.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (sidebar) | "New Canvas" |
| Empty state -- no file open (markdown panel) | Heading: "No file open" / Body: "Select a .md file from the sidebar to view it here." |
| Empty state -- no project files (sidebar) | Heading: "Empty project" / Body: "This folder has no files yet." |
| Empty state -- canvas loading (canvas panel) | No text -- show panel background color. TLDraw loads within ~1s; no loading spinner needed. |
| Error state -- file read failed (markdown) | "Could not read file" / "The file may have been moved or deleted. Check the sidebar and select another file." |
| Error state -- webview creation failed (canvas) | "Canvas unavailable" / "WebView could not be initialized. Try closing and reopening this panel." |
| Error state -- file watcher failed | No user-visible error. Log via tracing::warn. Markdown content remains static (stale but readable). |
| Panel title -- canvas | "Canvas" (static label, matching terminal's "Terminal" pattern) |
| Panel title -- markdown | File name without path, e.g. "README.md" (dynamic, updates on file change) |
| Panel title -- markdown (no file) | "Markdown" (default before a file is opened) |
| Sidebar section header | "FILES" (all caps, 11px semibold) |
| Sidebar "New Canvas" button | "New Canvas" (13px, accent-colored text, no button chrome) |
| Sidebar folder indicator | Prefix with unicode triangle: "\u{25B6} " (collapsed) / "\u{25BC} " (expanded) |

No destructive actions exist in Phase 3. Canvas auto-saves (D-02). Markdown is read-only (D-05). Panel close uses existing Phase 1 mechanism with no confirmation.

## Interaction Contracts

### Focus Routing (D-14, D-15, D-16)

| Trigger | Action | Visual Feedback |
|---------|--------|----------------|
| Click on GPU panel body | Set focused_panel, call focus_parent() if previous was webview | Remove overlay from clicked panel, apply overlay to previous |
| Click on webview panel body | Set focused_panel, call wv.focus() | Remove CSS desaturation from clicked, apply to previous; apply overlay to other GPU panels |
| Cmd+] | Cycle focus to next panel in grid order (wraps) | Same focus visuals as click |
| Cmd+[ | Cycle focus to previous panel in grid order (wraps) | Same focus visuals as click |
| Cmd+W (in webview) | JS intercepts, sends IPC shortcut message, Rust closes panel | Panel closes via existing mechanism |
| Cmd+B | Toggle sidebar visibility | Sidebar slides in/out (instant, no animation); grid recomputes width |

### Unfocused Panel Desaturation (D-16)

| Panel Type | Unfocused Treatment |
|------------|-------------------|
| GPU panel (terminal, markdown, sidebar) | Render a semi-transparent black overlay quad [0.0, 0.0, 0.0, 0.25] on top of all panel content |
| Webview panel (canvas) | Apply CSS `body.unfocused { filter: saturate(0.3) brightness(0.7) }` via evaluate_script |
| Focused panel | No overlay; full color rendering; CSS filter removed |

### File Sidebar (D-10 through D-13)

| Trigger | Action | Visual Feedback |
|---------|--------|----------------|
| Click file entry (.md) | Open in existing markdown panel or split focused panel (D-12) | Selected entry gets bg highlight + accent left bar |
| Click file entry (.tldr) | Open in existing canvas panel or split focused panel | Selected entry gets bg highlight + accent left bar |
| Click folder entry | Toggle expand/collapse | Triangle indicator rotates (\u{25B6} <-> \u{25BC}) |
| Hover file entry | Show hover background | Hover bg color applied |
| Click "New Canvas" | Create timestamped .tldr in .myco/canvas/, open it | New entry appears in sidebar, canvas panel opens |
| File system change (notify) | Refresh sidebar file tree | Entries add/remove with no animation |

### Markdown Viewer (D-04 through D-09)

| Trigger | Action | Visual Feedback |
|---------|--------|----------------|
| File opened from sidebar | Parse markdown, build styled blocks, render | Content appears in panel |
| Mouse wheel scroll | Scroll viewport offset up/down | Content scrolls; viewport-culled blocks update |
| File changed on disk (notify) | Re-parse markdown, rebuild blocks, re-render | Content updates in place (no flash or scroll reset) |
| Scroll to top/bottom | Clamp scroll position | No overscroll bounce |

### TLDraw Canvas (D-01 through D-03)

| Trigger | Action | Visual Feedback |
|---------|--------|----------------|
| Canvas panel created | Create wry webview, load bundled TLDraw HTML via myco:// protocol | TLDraw UI appears in panel |
| User draws/edits | TLDraw handles internally; store.listen fires after 1500ms debounce | No Myco-side visual feedback; TLDraw handles its own UI |
| Auto-save triggered | JS sends snapshot via IPC, Rust writes .tldr to .myco/canvas/ | No save indicator in Phase 3 (deferred to Phase 4 status bar) |
| Panel resized | Call wv.set_bounds(new_rect) | TLDraw reflows to new size |
| Canvas panel opened with existing .tldr | Rust reads file, sends load IPC message to JS | TLDraw restores previous state |

## Layout Contract

### Window Layout Structure

```
+----------------------------------------------------------+
|  Title Bar (38px, full width)                             |
+--------+-------------------------------------------------+
| Side-  |                                                  |
| bar    |  Grid Area (remaining width x remaining height)  |
| (240px |                                                  |
|  when  |  +--------------------+  +--------------------+  |
|  open, |  | Terminal (GPU)     |  | Markdown (GPU)     |  |
|  0px   |  |                    |  |                    |  |
|  when  |  +--------------------+  +--------------------+  |
|  closed)|  | Canvas (Webview)   |                         |
|        |  |                    |                         |
|        |  +--------------------+-------------------------+
+--------+-------------------------------------------------+
```

- Sidebar: Fixed 240px width, left edge, outside taffy grid. Background uses panel_background color.
- Grid: Fills `window_width - sidebar_width` (or full width when sidebar hidden) x `window_height - TITLE_BAR_HEIGHT`.
- Panel title bar: 28px height within each panel, contains type label and close/fullscreen buttons.
- Content area: Panel height minus 28px title bar.

### Markdown Viewer Internal Layout

```
+----------------------------------+
| Panel Title Bar (28px)           |
+----------------------------------+
| md:16px padding                  |
|                                  |
|  H1 (28px semibold, 32px top)   |
|  md:16px gap                     |
|  Body paragraph (15px, 24px lh)  |
|  md:16px gap                     |
|  H2 (22px semibold, 24px top)   |
|  md:16px gap                     |
|  Body paragraph                  |
|  md:16px gap                     |
|  +------------------------------+|
|  | Code block (14px mono)       ||
|  | lg:24px padding, code bg     ||
|  +------------------------------+|
|  md:16px gap                     |
|  | Blockquote border (3px)      ||
|  | sm:8px left padding          ||
|  | Body text (italic)           ||
|  md:16px gap                     |
|  --- HR (1px line, full width) --|
|                                  |
+----------------------------------+
```

Block spacing rules (vertical gap between consecutive blocks):
- After H1: 16px
- After H2: 16px
- After H3-H6: 16px
- After paragraph: 16px
- After code block: 16px
- After blockquote: 16px
- After HR: 16px
- Before H1: 32px (extra top margin)
- Before H2: 24px (extra top margin)
- Before H3: 16px

Content horizontal padding: 16px on left and right within the panel content area.
Maximum content width: None (fills panel width minus 32px total horizontal padding).

### File Sidebar Internal Layout

```
+------------------------------+
| md:16px top padding          |
| "FILES" heading (11px semi)  |
| sm:8px gap                   |
| > folder_name/ (13px)       |  <- 0px indent
|   file.rs (13px)            |  <- 16px indent (1 level)
|   subfolder/ (13px)         |  <- 16px indent
|     deep.rs (13px)          |  <- 32px indent (2 levels)
| file_at_root.md (13px)      |  <- 0px indent
| sm:8px gap                   |
| "New Canvas" button (13px)   |
+------------------------------+
```

- Each file entry: 28px height, 16px left padding + (depth * 16px) indent
- Folder expand triangle: inline with text, no extra width
- Selected item: full-width bg highlight (sidebar_selected_bg) with 2px accent bar on left edge
- Hover item: full-width bg highlight (sidebar_hover_bg)
- Scrolling: Mouse wheel scrolls sidebar independently of grid panels

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none | not applicable (Rust project, no web component library) |
| npm (TLDraw bundle) | tldraw@5.0.1, react@18, react-dom@18 | Bundled at build time as static assets; no runtime npm access; TLDraw is MIT licensed |

TLDraw JS/CSS is bundled as app resources (D-01). The TLDraw Vite build output is committed to the repository or built during CI. No runtime package installation. No third-party shadcn registries.

**npm dependency note:** TLDraw and React are consumed via a one-time Vite production build that produces static JS/CSS files. These files are served via wry's custom protocol handler (`myco://`). No Node.js runtime is required at application runtime.
