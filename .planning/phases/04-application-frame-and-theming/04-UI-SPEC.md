# Phase 4 — UI Design Contract

> Visual and interaction contract for the application frame, status bars, settings overlay, and theme system. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Rust + wgpu GPU rendering, not a web stack) |
| Preset | not applicable |
| Component library | Custom GPU-rendered (quad_renderer + glyphon text) |
| Icon library | Unicode glyphs (text-rendered, no external icon font) |
| Font | System fonts via cosmic-text FontSystem (SF Pro on macOS, monospace: JetBrains Mono/Fira Code configurable) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, accent bar width (2px is exception) |
| sm | 8px | Compact element spacing, panel content padding, stats slot padding |
| md | 16px | Default element spacing, paragraph gaps, content padding |
| lg | 24px | Section padding, code block padding, heading margins |
| xl | 32px | Large heading top margins |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Not used in this phase |

Exceptions:
- 2px: accent left bar in sidebar selection, status indicator dots
- 28px: panel title bar height (PANEL_TITLE_HEIGHT, established Phase 1)
- 38px: app title bar height (TITLE_BAR_HEIGHT, established Phase 1)
- 240px: sidebar width (SIDEBAR_WIDTH, established Phase 3)

These are the non-grid fixed regions that compose the application frame:

| Region | Height | Position | Content |
|--------|--------|----------|---------|

Window layout deduction order:
1. Title bar: 38px from top
2. Stats bar: 24px below title bar
3. Bottom bar: 24px from bottom
4. Sidebar: 240px from left (if visible)
5. Remaining space: grid layout

## Color — Theme Architecture

### Layered Definition (D-11)

Each theme defines 8-10 base semantic colors. The full palette is derived from these:

| Base Slot | Semantic Role | Dracula Value | Derivation |
|-----------|--------------|---------------|------------|
| `bg_primary` | Window/app background | #282A36 | Direct |
| `bg_secondary` | Panel background, sidebar | #2C2E3B | Derived: lighten bg_primary 3% |
| `bg_tertiary` | Selection, hover highlight, current line | #44475A | Derived: lighten bg_primary 12% |
| `fg_primary` | Body text, primary content | #F8F8F2 | Direct |
| `fg_secondary` | Comments, muted text, folder names | #6272A4 | Direct |
| `accent` | Active elements, links, divider hover | #BD93F9 | Direct (purple for Dracula) |
| `success` | Running indicator, green terminal | #50FA7B | Direct |
| `warning` | Attention states | #FFB86C | Direct |
| `error` | Destructive, errors | #FF5555 | Direct |
| `border` | Dividers, separators | #353748 | Derived: between bg_primary and bg_tertiary |

### Derived Colors (auto-computed from base slots)

| Derived Field | Source | Computation |
|---------------|--------|-------------|
| `title_bar_text` | fg_primary | Same value |
| `panel_label_text` | fg_secondary | Same value |
| `divider` | border | Same value |
| `divider_hover` | accent | Same value |
| `sidebar_selected_bg` | bg_tertiary | Same value |
| `sidebar_hover_bg` | border | Same value |
| `sidebar_folder_text` | fg_secondary | Same value |
| `markdown_heading_text` | accent | Same value |
| `markdown_body_text` | fg_primary | Same value |
| `markdown_code_text` | success | Same value |
| `markdown_code_block_bg` | bg_primary darkened 5% | Computation |
| `markdown_blockquote_border` | fg_secondary | Same value |
| `markdown_link_text` | theme-specific (cyan) | Override allowed |
| `markdown_hr` | bg_tertiary | Same value |
| `unfocused_overlay` | [0, 0, 0, 0.25] | Fixed |

### 60/30/10 Distribution (Dracula default)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #282A36 | Window background, panel backgrounds, title bars |
| Secondary (30%) | #44475A / #353748 | Selection highlights, sidebar hover, dividers, code blocks |
| Accent (10%) | #BD93F9 | Sidebar active indicator, divider hover, markdown headings, focused panel ring |

Accent reserved for:
- Sidebar left-edge active indicator (2px bar)
- Divider hover highlight
- Markdown heading color
- Settings active/selected item
- Stats bar active slot underline

## Theme JSON Schema (D-15)

Custom themes stored at `~/.myco/themes/{name}.json`:

```json
{
  "name": "My Custom Theme",
  "base": {
    "bg_primary": "#282A36",
    "bg_secondary": "#2C2E3B",
    "bg_tertiary": "#44475A",
    "fg_primary": "#F8F8F2",
    "fg_secondary": "#6272A4",
    "accent": "#BD93F9",
    "success": "#50FA7B",
    "warning": "#FFB86C",
    "error": "#FF5555",
    "border": "#353748"
  },
  "ansi": {
    "colors": ["#000000", "#FF5555", "#50FA7B", "#F1FA8C", "#BD93F9", "#FF79C6", "#8BE9FD", "#BBBBBB",
               "#555555", "#FF5555", "#50FA7B", "#F1FA8C", "#CAA9FA", "#FF79C6", "#8BE9FD", "#FFFFFF"],
    "foreground": "#F8F8F2",
    "background": "#282A36"
  },
  "overrides": {
    "markdown_link_text": "#8BE9FD"
  }
}
```

Fields:
- `base` (required): The 10 base colors. All derived colors auto-computed.
- `ansi` (required): 16-color terminal palette + fg/bg.
- `overrides` (optional): Override any specific derived color by field name.

## State Transitions

### Theme Switch

1. User selects new theme from Settings dropdown
2. App replaces `self.theme` with new Theme instance
3. App updates AnsiPalette for all terminal instances
4. App requests full window redraw (request_redraw)
5. All GPU-rendered content uses new colors on next frame
6. Webviews are NOT re-themed (they have independent styling)

### Project Switch (D-04)

1. User clicks different project in switcher
2. Current state: close all PTYs, destroy webviews, clear grid panels
3. Load new project's .myco config (or defaults if none)
4. Rebuild sidebar file tree for new project_dir
5. Rebuild grid layout with new panel configuration
6. Bottom bar updates git branch/path
7. Stats bar resets uptime counter

### Settings Open/Close

1. Cmd+, sets `app_state = AppState::Settings`
2. Render loop skips grid rendering, renders settings overlay instead
3. Input routes to settings UI (keyboard nav, mouse clicks on controls)
4. Esc sets `app_state = AppState::Workspace`
5. Render loop returns to normal grid rendering
6. All panels preserved underneath (same as fullscreen mechanism)

## Implementation Notes

### Color Space

All theme colors stored as sRGB hex strings in JSON configuration. Converted to linear-light `[f32; 4]` at load time for wgpu rendering (wgpu uses linear-light color space on sRGB surfaces). The existing `linear_to_srgb_u8` helper is used when passing colors to glyphon (which expects sRGB u8 values).

Conversion function needed: `hex_to_linear(hex: &str) -> [f32; 4]` (parse hex, convert sRGB to linear-light).

### Theme Struct Refactor

Current `Theme` struct has ~20 explicit fields. Refactor to:
1. `ThemeBase` struct: 10 base color slots
2. `ThemeDerived` struct: all computed colors (from base + optional overrides)
3. `ThemeDefinition`: base + ansi + overrides (what's stored in JSON)
4. `Theme`: fully resolved (ThemeDerived + AnsiPalette), ready for rendering

The `Theme` struct passed to renderers remains flat (direct field access for performance), but it is constructed from the layered definition.

### Stats Bar Slot Architecture

```rust
struct StatsSlot {
    label: String,    // e.g. "Panels"
    value: String,    // e.g. "4"
    visible: bool,
}
```

Stats bar renders only visible slots. Phase 6 adds new slot types (token usage, LLM status) without changing the rendering logic.
