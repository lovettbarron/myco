In src/app.rs, modify the workspace keyboard action dispatch loop (currently at lines 3397-3399). The current code is:

```rust
for action in actions {
    self.process_action(action);
}
```

Replace it with quit-aware handling that mirrors the CloseRequested save-on-exit pattern (lines 2905-2919):

```rust
for action in actions {
    if matches!(action, InputAction::Quit) {
        // Save config on quit (same logic as CloseRequested)
        if let (Some(grid), Some(project_dir)) =
            (&self.grid, &self.project_dir)
        {
            let config =
                crate::config::ProjectConfig::from_current_state(
                    grid,
                    &self.panels,
                    self.terminal_manager.as_ref(),
                    project_dir,
                    Some(&self.theme_registry.active().name),
                );
            crate::config::save_project_config(
                project_dir,
                &config,
            );
            info!("Saved project config on quit");
        }
        info!("Quit requested via shortcut -- exiting");
        event_loop.exit();
        return;
    }
    self.process_action(action);
}
```

Key points:
- `event_loop` is available in `window_event` scope (parameter on line 2888) -- this is why the quit action is handled here and not in `process_action` (which has no event_loop parameter).
- The save pattern is identical to CloseRequested (lines 2905-2919): build ProjectConfig from current state, write to disk, then exit.
- `return` after `event_loop.exit()` prevents processing any remaining actions in the vec.
- The picker mode hardcoded Cmd+Q (line 3269-3272) remains untouched -- it executes before the workspace keyboard path and does not need the save logic (picker mode has no open project layout to save).

Do NOT modify process_action -- it intentionally lacks event_loop access. The quit action is intercepted before reaching process_action, consistent with the picker mode pattern where Cmd+Q is also handled directly in the window_event method.
