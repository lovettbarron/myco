**src/config/registry.rs** — Per D-09, D-11, D-12:
```rust
use std::path::{Path, PathBuf};
use serde::{Deserialize, Serialize};
use tracing::warn;

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ProjectEntry {
    pub path: PathBuf,
    pub name: String,
    #[serde(skip_serializing_if = "Option::is_none")]
    pub last_opened: Option<String>,  // ISO 8601 timestamp
}

impl ProjectEntry {
    pub fn exists(&self) -> bool {
        self.path.exists()
    }
}

#[derive(Debug, Clone, Serialize, Deserialize)]
struct RegistryFile {
    version: u32,
    projects: Vec<ProjectEntry>,
}

pub struct ProjectRegistry {
    pub projects: Vec<ProjectEntry>,
    path: Option<PathBuf>,
}
```

Methods:
- `pub fn new() -> Self` — calls `Self::load()`
- `pub fn load() -> Self` — reads `~/.myco/projects.json` via dirs::home_dir(). File size cap 1MB. Returns empty on error.
- `pub fn save(&self)` — atomic write to `~/.myco/projects.json`
- `pub fn register(&mut self, project_path: &Path)` — if path not already in list, add new ProjectEntry { path, name: folder name, last_opened: now ISO 8601 }. If already exists, update last_opened. Then save.
- `pub fn remove(&mut self, project_path: &Path)` — remove entry with matching path, then save.
- `pub fn update_last_opened(&mut self, project_path: &Path)` — update timestamp, save.

app_state: AppState::Picker,  // Default to picker; overridden if CLI arg

4. Modify `App::resumed()`:
- At the very beginning (after window creation, renderer init, font loading), check for CLI argument:
  ```rust
  let cli_project_dir = std::env::args().nth(1).map(PathBuf::from);
  ```
- If `cli_project_dir` is Some and the path exists (D-10: skip picker):
  - Set `self.app_state = AppState::Workspace;`
  - Proceed with existing workspace initialization (load config, build grid, etc.)
  - Call `self.project_registry.register(&project_dir);` to auto-register (D-11)
- If `cli_project_dir` is None (D-09: show picker):
  - Set `self.app_state = AppState::Picker;`
  - Create PickerState from registry: `self.picker_state = Some(PickerState::new(self.project_registry.projects.clone()));`
  - Do NOT initialize grid, terminal_manager, sidebar, etc. yet
  - Store window and renderer only
  - Return early (don't run the workspace setup code)

5. Add `fn open_project(&mut self, project_path: PathBuf)`:
- Transitions from Picker to Workspace
- Sets `self.app_state = AppState::Workspace;`
- Sets `self.project_dir = Some(project_path.clone());`
- Registers in registry: `self.project_registry.register(&project_path);`
- Runs the workspace initialization code (moved from resumed() into a helper):
  - Load config, build grid from config (or default), create panels, spawn terminals/canvases
  - Initialize sidebar, bottom_bar, file_watcher
- Clears picker_state: `self.picker_state = None;`

6. Modify render loop (in RedrawRequested handler):
- If `self.app_state == AppState::Picker`:
  - Build quads + labels from picker renderer
  - Render them (same pipeline as settings overlay)
  - Skip workspace rendering entirely
- If `self.app_state == AppState::Workspace`:
  - Existing workspace rendering (grid, panels, terminal, etc.)

7. Modify input handling (KeyboardInput, MouseInput, CursorMoved):
- If `self.app_state == AppState::Picker`:
  - Route keyboard to picker (Enter = open selected, Up/Down = navigate, Escape = quit, Cmd+O = folder dialog)
  - Route mouse clicks to `picker_state.handle_click()`
  - Route mouse move to `picker_state.hovered` update
  - On PickerAction::OpenProject(path) → call `self.open_project(path)`
  - On PickerAction::OpenFolderDialog → use rfd::FileDialog or native NSOpenPanel (defer to checkpoint if complex; for now just open CWD)
- If `self.app_state == AppState::Workspace`:
  - Existing input handling
