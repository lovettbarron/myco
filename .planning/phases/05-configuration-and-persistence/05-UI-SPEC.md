# Phase 5 -- UI Design Contract

> Visual and interaction contract for configuration persistence, project picker, keyboard shortcut customization, and layout save/restore. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Rust + wgpu GPU rendering, not a web stack) |
| Preset | not applicable |
| Component library | Custom GPU-rendered (quad_renderer + glyphon text) |

## State Transitions

### Application Launch (D-09, D-10)

```
Launch with CLI path argument:
  1. Skip project picker
  2. Auto-register project in ~/.myco/projects.json (D-11)
  3. Load .myco/config.json (or create defaults)
  4. Restore layout from config (D-05)
  5. Render workspace

Launch without CLI argument:
  1. Read ~/.myco/projects.json
  2. Render project picker (fullscreen)
  3. User selects project -> go to step 3 above
  4. User clicks "Open Folder..." -> folder dialog -> go to step 2 (auto-register) above
```

### Layout Save (D-07, D-08)

```
Trigger: structural change (panel split, close, cap type change, file navigation)
  1. Start 2-second debounce timer
  2. If another change arrives within 2 seconds, reset timer
  3. On timer expiry: serialize current layout to .myco/config.json
  4. Serialization captures: grid structure, cap types, file references (relative paths per D-04)
  5. Terminal caps save CWD. Canvas caps save .tldr path. Markdown caps save .md path.
  6. Full file overwrite (D-08), not patch
```

### Layout Restore (D-04, D-05, D-06)

```
On project open:
  1. Read .myco/config.json
  2. Parse grid layout (panel arrangement, cap types)
  3. Rebuild taffy tree with equal splits (D-06: no exact ratios in v1)
  4. For each cap:
     - Terminal: spawn new PTY at saved CWD
     - Canvas: create webview, load saved .tldr file
     - Markdown: load saved .md file, start file watcher
  5. If config missing or malformed: create default layout (single terminal panel)
```

### Shortcut Rebinding (D-14 to D-18)

```
User clicks shortcut row:
  1. Enter key-recording mode (visual: pulsing accent border)
  2. Capture first key combo (modifiers + key)
  3. Display first key-badge
  4. Start 1-second chord timeout
  5a. If second key combo within timeout: record as chord, save
  5b. If timeout expires: record as single combo, save
  6. Check conflicts against all other bindings
  7. If conflict: override silently, show notification toast with "Undo" link
  8. Write sparse overrides to ~/.myco/shortcuts.json (D-18)
  9. Exit recording mode, display new badge(s)

Undo flow (from notification toast):
  7a. User clicks "Undo" within 3-second auto-dismiss window
  7b. Restore previous binding for the displaced action
  7c. Remove the new binding (revert to previous state for both actions)
  7d. Dismiss notification immediately
  7e. Re-write ~/.myco/shortcuts.json with restored state
```

### Project Switch (sidebar)

```
User clicks different project in sidebar switcher:
  1. Auto-save current layout (immediate, not debounced)
  2. Close all PTYs
  3. Destroy all webviews
  4. Clear grid panels
  5. Update project_dir
  6. Load new project's .myco/config.json
  7. Restore layout (or defaults)
  8. Refresh sidebar file tree
  9. Update bottom bar (git branch, path)
  10. Reset stats bar uptime
```

## Config File Schemas

### .myco/config.json (D-03, D-04, D-05)

```json
{
  "version": 1,
  "project": {
    "name": "myco",
    "description": ""
  },
  "theme": null,
  "layout": {
    "panels": [
      {
        "type": "terminal",
        "cwd": "."
      },
      {
        "type": "canvas",
        "file": "sketches/plan.tldr"
      },
      {
        "type": "markdown",
        "file": "README.md"
      }
    ],
    "grid": {
      "columns": 2,
      "rows": 1,
      "splits": []
    }
  }
}
```

Fields:
- `version` (required): Schema version for future migration
- `project.name`: Display name (derived from folder name if absent)
- `project.description`: Optional description
- `theme`: Theme name override (null = use global default, D-01)
- `layout.panels`: Ordered list of caps with type and file references (relative paths, D-04)
- `layout.grid`: Grid structure (columns, rows, nested splits)

### ~/.myco/projects.json (D-09, D-11, D-12)

```json
{
  "version": 1,
  "projects": [
    {
      "path": "/Users/andrew/src/myco",
      "name": "myco",
      "last_opened": "2026-05-17T12:00:00Z"
    }
  ]
}
```

### ~/.myco/shortcuts.json (D-17, D-18)

```json
{
  "version": 1,
  "bindings": [
    {
      "action": "panel_split_h",
      "keys": ["cmd+d"]
    },
    {
      "action": "focus_next_panel",
      "keys": ["cmd+]"]
    },
    {
      "action": "some_chord_action",
      "keys": ["cmd+k", "cmd+s"]
    }
  ]
}
```

Sparse format (D-18): only overridden bindings stored. Missing actions use built-in defaults.

## Notification Toast (conflict resolution, D-16)

| Element | Spec |
|---------|------|
| Position | Bottom-right corner, 16px from edges |
| Width | 280px |
| Height | Auto (content-driven), min 40px |
| Background | bg_secondary |

## Implementation Notes

### Config Directory Bootstrap

Extend existing `src/context.rs` `ensure_context_files()` to also create `.myco/config.json` with default layout when a project is first opened. Pattern already established for `.myco/context/` directory.

### Shortcut System Architecture

Replace hardcoded match arms in `src/input/keyboard.rs` with a lookup table:

```rust
struct ShortcutRegistry {
    bindings: HashMap<KeyCombo, InputAction>,
    chord_state: Option<(KeyCombo, Instant)>, // first key of chord + timestamp
}
```

The existing `InputAction` enum already covers all bindable actions. The registry loads built-in defaults, then overlays user overrides from `~/.myco/shortcuts.json`.

### Debounce Timer for Auto-Save

Reuse the pattern from `notify-debouncer-full` (already in the dependency stack). A 2-second debounce after any structural grid change before writing `.myco/config.json`.

### Layout Serialization

Convert taffy tree structure to the flat `layout.panels[]` + `layout.grid` schema. On restore, rebuild the taffy tree with `Style::default()` flex values (equal splits per D-06).
