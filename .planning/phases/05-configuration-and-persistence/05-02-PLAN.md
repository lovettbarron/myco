**src/shortcuts/chord.rs** — Per D-15 chord support:
- `const CHORD_TIMEOUT: Duration = Duration::from_millis(500);`
- `#[derive(Debug, Clone, PartialEq, Eq, Hash)] pub struct KeyCombo { pub key: String, pub modifiers: Modifiers }`
- `#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Default)] pub struct Modifiers { pub cmd: bool, pub ctrl: bool, pub shift: bool, pub alt: bool }`
- Implement `Modifiers::cmd()`, `Modifiers::ctrl()`, etc. convenience constructors
- `pub fn parse_key_string(s: &str) -> KeyCombo` — splits on "+", identifies modifiers (cmd/super/meta, ctrl/control, shift, alt/option), remaining part is the key
- `pub fn key_combo_to_string(combo: &KeyCombo) -> String` — reverse of parse_key_string
- `pub enum ChordState { Idle, Pending { prefix: KeyCombo, started: Instant } }`
- `pub enum ResolveResult { Action(String), Pending, NoMatch, Timeout }`
- `pub struct ChordStateMachine { state: ChordState }`
- `ChordStateMachine::new()` starts Idle
- `ChordStateMachine::feed(&mut self, combo: &KeyCombo, registry: &ShortcutRegistry) -> ResolveResult`:
  - If Idle: check if combo matches any single-key binding → Action. Check if combo is a prefix of any chord → Pending (set started=Instant::now()). Otherwise NoMatch.
  - If Pending: combine prefix + combo as chord, lookup in registry → Action if found, otherwise reset to Idle and try combo as single-key.
- `ChordStateMachine::check_timeout(&mut self) -> bool` — returns true if in Pending state and elapsed > CHORD_TIMEOUT, resets to Idle
- `ChordStateMachine::reset(&mut self)` — force back to Idle
- `pub fn key_combo_from_event(event: &KeyEvent, modifiers: &ModifiersState) -> Option<KeyCombo>` — converts winit KeyEvent + ModifiersState to KeyCombo. Handles Key::Character (lowercase) and Key::Named (escape, enter, tab, backspace, arrows, f1-f12). Returns None for unrecognized keys.
