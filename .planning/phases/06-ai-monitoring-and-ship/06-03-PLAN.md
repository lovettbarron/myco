Intervention detection that surfaces toast notifications when terminal processes need human attention.

Purpose: When Claude Code asks a permission question or sudo requests a password, the user gets a toast notification immediately — no need to be actively watching that terminal. This is the "AI-native" feature that makes Myco aware of what's happening in its terminal panels.

Output: Background scanner reads terminal grid text, matches against patterns, creates intervention toasts. Clicking toast focuses source panel. Dismissing suppresses the pattern for the session.

    ```rust
    fn extract_terminal_visible_text(term: &Arc<FairMutex<Term<impl EventListener>>>) -> String {
        let term = term.lock();
        let grid = term.grid();
        let cols = grid.columns();
        let screen_lines = grid.screen_lines();
        // Per Claude's Discretion: scan last visible screen area only
        let mut text = String::with_capacity(cols * screen_lines);
        for line_idx in 0..screen_lines {
            let row = &grid[alacritty_terminal::index::Line(line_idx as i32)];
            for col in 0..cols {
                text.push(row[alacritty_terminal::index::Column(col)].c);
            }
            text.push('\n');
        }
        drop(term); // Release lock immediately (Pitfall 4: FairMutex contention)
        text
    }
    ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Denial of Service | intervention.rs | mitigate | Substring matching only (no regex engine). Matchers capped at 200 chars. Max 100 patterns. Scan rate limited to once per 2s per panel. |
| T-06-03 | Denial of Service | toast/mod.rs | mitigate | Max 3 visible toasts. Rate limit 1 per pattern per panel per 10s (enforced in ToastManager::add). |
| T-06-04 | Information Disclosure | patterns.rs | mitigate | Fixed path ~/.myco/patterns.json only. Path never constructed from terminal output or user input. File size capped at 1MB. |
</threat_model>
