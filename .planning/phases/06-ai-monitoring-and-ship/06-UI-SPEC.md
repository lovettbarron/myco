# Phase 6 — UI Design Contract

> Visual and interaction contract for the AI Monitoring and Ship phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (GPU-rendered via wgpu + QuadInstance + glyphon) |
| Preset | not applicable |
| Component library | none (custom GPU primitives: QuadInstance, TextLabel, overlays) |
| Icon library | none (text glyphs used as icons: "x" for close, Unicode symbols for indicators) |
| Font | System default via cosmic-text (JetBrains Mono configurable for terminal) |

---

## Spacing Scale

Declared values (must be multiples of 4). Derived from existing codebase constants.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element margins (close button y-offset = 6px is closest) |
| sm | 8px | Panel content padding (PANEL_CONTENT_PADDING), toast stack gap, text insets |
| md | 16px | Toast margin from viewport edge, tooltip padding, context menu item height |
| lg | 24px | Stats bar height (STATS_BAR_HEIGHT), bottom bar height (BOTTOM_BAR_HEIGHT) |
| xl | 28px | Panel title bar height (PANEL_TITLE_HEIGHT) |
| 2xl | 48px | Toast height (existing pattern from settings.rs) |

Exceptions:
- Resource dot diameter: 8px (centered vertically in 28px panel header)
- Tooltip pointer offset: 4px below dot
- Context menu item height: 28px (matches panel title bar for visual consistency)
- Close/fullscreen button size: 16x16px (existing pattern)

---

## Typography

All text rendered via glyphon + cosmic-text. Font sizes in CSS px (converted to cosmic-text metrics internally).

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 13px | 400 (regular) | 1.4 | Toast message text, tooltip stats text, context menu items |
| Label | 12px | 400 (regular) | 1.3 | Panel title bar text (existing), tooltip secondary labels |
| Small | 11px | 400 (regular) | 1.2 | Toast secondary text (e.g. panel name attribution), stat units |
| Heading | 14px | 600 (semibold) | 1.2 | Tooltip header ("Panel Resources"), context menu section headers (if any) |

Note: These match exactly the font sizes already used in the codebase (app.rs uses 11, 12, 13, 14; status_bar.rs uses 11, 13).

## Color

Colors are theme-derived. This contract specifies which existing `Theme` struct fields map to each new UI element. All colors reference `ThemeBase` semantic names so they adapt across Dracula, Solarized, and Obsidian themes.

| Role | Theme Field | Dracula Hex | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | `background` / `panel_background` | #282A36 / #2C2E3B | Window bg, panel body, tooltip bg |
| Secondary (30%) | `bg_secondary` | #2C2E3B | Toast background, context menu background, tooltip background |
| Accent (10%) | `divider_hover` (accent) | #BD93F9 | Toast left accent bar, "Focus" link in intervention toast, active context menu item highlight |
| Destructive | `error` (ThemeBase) | #FF5555 | Resource dot red state (>100% CPU), error toast accent bar |

### Resource Dot Colors (new semantic mappings)

| State | Theme Field | Dracula Hex | Condition |
|-------|-------------|-------------|-----------|
| Healthy | `success` | #50FA7B | CPU < 50% single-core |
| Elevated | `warning` | #FFB86C | CPU 50-100% single-core |
| Critical | `error` | #FF5555 | CPU > 100% (multi-core saturation) |
| No process | `fg_secondary` | #6272A4 | No associated process (e.g. empty panel) |

### Frozen Panel Overlay (new)

| Element | Color Value | Description |
|---------|-------------|-------------|
| Overlay | `[0.1, 0.2, 0.4, 0.35]` (linear RGBA) | Blue-tinted semi-transparent overlay covering entire panel rect |
| Frozen icon | `fg_primary` with 0.8 alpha | Snowflake/pause Unicode glyph in panel header |

Accent reserved for: toast left accent bar, "Focus" link text in intervention toasts, active/hovered context menu item highlight, frozen panel header icon glow. Never used for resource dots (those use success/warning/error semantic colors).

## Interaction Contracts

### Resource Dot Hover -> Tooltip

1. Mouse enters 8x8 resource dot hit area (expand to 16x16 for easier targeting)
2. After 300ms hover delay, tooltip appears below dot
3. Tooltip shows CPU % and RAM MB from last poll (max 2s stale)
4. Mouse leaves both dot and tooltip -> tooltip disappears immediately (no fade)
5. If mouse moves directly from dot into tooltip, tooltip stays visible

### Right-Click -> Context Menu

1. User right-clicks (or Ctrl+Click on macOS) anywhere in the 28px panel header
2. Native NSMenu appears at click coordinates
3. Menu items depend on panel state (frozen/unfrozen, has process/no process)
4. User selects item -> InputAction dispatched (FreezePanel/UnfreezePanel/PanelClose)
5. Menu dismissed on click-outside (native behavior)

### Freeze -> Overlay

1. FreezePanel action dispatched
2. For terminal panels: SIGSTOP sent to child process tree
3. `panel.frozen = true` set immediately
4. Blue overlay quad rendered over panel on next frame
5. Snowflake/pause glyph appended to panel title on next frame
6. Resource dot continues updating (process still exists, just stopped)
7. All input to panel body blocked; only header right-click responds

### Unfreeze -> Resume

1. UnfreezePanel action dispatched
2. For terminal panels: SIGCONT sent to child process tree
3. `panel.frozen = false` set immediately
4. Blue overlay removed on next frame
5. Snowflake/pause glyph removed from title on next frame
6. Input routing restored

### Intervention Detection -> Toast

1. Background scanner checks last visible screen area of each terminal panel every 2s
2. Pattern match found against known patterns (built-in + ~/.myco/patterns.json)
3. Toast created: type=Intervention, message from pattern config
4. Toast rendered in bottom-right stack
5. After 8 seconds: toast auto-dismissed. Resource dot remains as secondary indicator.
6. User clicks "Focus" or toast body -> source panel focused, toast dismissed
7. User dismisses without focusing -> pattern suppressed for this terminal session

## Theme Integration Notes

All new UI elements MUST derive colors from the existing `Theme` struct fields. New fields needed on the `Theme` struct:

| New Field | Derived From | Purpose |
|-----------|-------------|---------|
| `frozen_overlay` | Hardcoded `[0.1, 0.2, 0.4, 0.35]` | Blue-tinted frozen panel overlay |
| `resource_healthy` | `success` | Green resource dot |
| `resource_elevated` | `warning` | Yellow resource dot |
| `resource_critical` | `error` | Red resource dot |
| `tooltip_bg` | `bg_secondary` | Tooltip background |

These can be added as direct fields on `Theme` with derivation in `Theme::from_definition()`, or as overridable fields in `ThemeDefinition::overrides`. Recommended: add as direct fields with `from_definition` defaults matching the semantic colors above, overridable via the existing `overrides` HashMap.
