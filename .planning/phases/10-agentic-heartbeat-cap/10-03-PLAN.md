2. Create src/heartbeat/scheduler.rs:
   - Imports: use super types, std::sync::mpsc, std::thread, std::time::{Duration, Instant}, tracing::{debug, info, warn}
   - Constants:
     ```rust
     const TICK_INTERVAL: Duration = Duration::from_secs(1);
     const INITIAL_BACKOFF: Duration = Duration::from_secs(5);
     const MAX_BACKOFF: Duration = Duration::from_secs(300);
     const BACKOFF_MULTIPLIER: f64 = 2.0;
     const HTTP_TIMEOUT: Duration = Duration::from_secs(120);
     ```
   - `fn next_backoff(current: Duration) -> Duration` -- `min(current * BACKOFF_MULTIPLIER, MAX_BACKOFF)`
   - `HeartbeatScheduler` struct:
     ```rust
     pub struct HeartbeatScheduler {
         command_sender: mpsc::Sender<SchedulerCommand>,
         _handle: std::thread::JoinHandle<()>,
     }
     ```
   - `HeartbeatScheduler::new(event_sender: std::sync::mpsc::Sender<HeartbeatEvent>, project_dir: PathBuf, llm_config: LlmConfig) -> Self`:
     - Create `mpsc::channel::<SchedulerCommand>()`
     - Spawn named thread `"heartbeat-scheduler"` via `std::thread::Builder::new().name(...)`
     - Thread body maintains:
       - `jobs: Vec<HeartbeatJob>` -- current job list
       - `last_run: HashMap<String, Instant>` -- when each job last executed
       - `client: reqwest::blocking::Client` with `timeout(HTTP_TIMEOUT)`
       - `provider: Option<LlmProvider>` -- built from LlmConfig, rebuilt on UpdateConfig
       - `backoff: Option<Duration>` -- current backoff if provider unreachable
       - `last_health_check: Instant` -- for backoff timing
       - `llm_config: LlmConfig` -- current config
       - `concurrency_slots: usize` -- from llm_config.heartbeat_concurrency (default 1)
       - `currently_running: usize` -- track active jobs (for single-threaded scheduler, always 0 or 1)
     - Thread loop (runs every TICK_INTERVAL):
       1. Non-blocking command check: `while let Ok(cmd) = cmd_rx.try_recv() { match cmd { ... } }`
         - `Shutdown` => return (exit thread)
         - `RunNow(name)` => set `run_now.insert(name)` flag
         - `ReloadJobs(new_jobs)` => replace `jobs = new_jobs`
         - `UpdateConfig(cfg)` => rebuild provider from new config, update concurrency
       2. If in backoff mode: check if backoff duration elapsed since last_health_check. If elapsed, probe Ollama health. If healthy, clear backoff and set `last_health_check = Instant::now()`. Send `HeartbeatEvent::HealthChanged { provider_healthy: true/false }`.
       3. For each job in `jobs`:
         - Skip if `!job.enabled`
         - Skip if `currently_running >= concurrency_slots`
         - Check if due: `run_now.remove(&job.name)` is true, OR interval elapsed (compare `last_run[job.name]` against `job.schedule.interval_minutes`)
         - If due, execute:
           a. Send `HeartbeatEvent::JobStarted { job_name }`
           b. Resolve files: `prompt::assemble_file_contents(&project_dir, &job.files, job.max_files, job.max_bytes)`
           c. Resolve template: `prompt::resolve_template(&job.prompt, &file_contents, &file_list, &project_name, file_count)`
           d. Determine provider: job.provider_override/model_override if set, else default from config
           e. Call `provider.generate(&client, &resolved_prompt)`
           f. On success: parse severity from response, build HeartbeatResult, save to disk via `config::save_result`, enforce retention via `config::enforce_retention`
           g. Send `HeartbeatEvent::JobCompleted { result }`
           h. On error: send `HeartbeatEvent::JobFailed { job_name, error }`. If connection error, enter backoff mode.
           i. Update `last_run.insert(job.name.clone(), Instant::now())`
       4. `std::thread::sleep(TICK_INTERVAL)`
   - `HeartbeatScheduler::send_command(&self, cmd: SchedulerCommand)` -- sends via command_sender, logs warning on failure
   - `HeartbeatScheduler::run_now(&self, job_name: String)` -- convenience for SendCommand(RunNow(name))
   - `HeartbeatScheduler::reload_jobs(&self, jobs: Vec<HeartbeatJob>)` -- convenience
   - `HeartbeatScheduler::shutdown(&self)` -- sends Shutdown command

   Note on thread communication: The scheduler uses `mpsc::Sender<HeartbeatEvent>` (not `EventLoopProxy<UserEvent>`) to send results. The main thread will bridge this to the winit event loop in Plan 05. This avoids a dependency on winit types in the scheduler module.
