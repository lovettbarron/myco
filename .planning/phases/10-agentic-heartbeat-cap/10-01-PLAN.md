<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-authored job JSON -> application | Untrusted JSON files from .myco/heartbeats/ deserialized into HeartbeatJob structs |
| Application -> Ollama API | Local HTTP to potentially attacker-controlled endpoint if endpoint overridden |
| Application -> Anthropic API | External HTTPS with API key in header |
| LLM response -> application | Untrusted text from LLM parsed for severity tags |
| File glob -> filesystem | Glob patterns could resolve to files outside project directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Information Disclosure | llm_client.rs | mitigate | Never log API key values. Use `tracing::debug` for request URLs but redact x-api-key header value. API key resolved from env var first (D-11) so it never needs to be in config file. |
| T-10-02 | Tampering | config.rs | mitigate | Validate job JSON: MAX_JOB_FILE_SIZE (1MB), MAX_JOBS (50), MAX_PROMPT_LEN (10000), MAX_FILE_PATTERNS (50). Reject oversized/malformed files with tracing::warn. |
| T-10-03 | Tampering | prompt.rs | mitigate | Validate glob-resolved paths start with project_dir using Path::starts_with(). Skip files outside project boundary. Log skipped paths with tracing::warn. |
| T-10-04 | Denial of Service | prompt.rs | mitigate | Enforce max_files (default 50) and max_bytes (default 100KB) limits per job. Break iteration when limits reached. |
| T-10-05 | Spoofing | llm_client.rs | accept | SSRF risk from custom Ollama endpoint: default is localhost:11434, user-configured endpoints are trusted because user controls their own config. Log non-localhost endpoints at warn level. |
| T-10-06 | Information Disclosure | global.rs | mitigate | LlmConfig in preferences.json should never store API keys. Document that ANTHROPIC_API_KEY comes from env var only. If user puts key in config, it works but is their choice. |
</threat_model>
