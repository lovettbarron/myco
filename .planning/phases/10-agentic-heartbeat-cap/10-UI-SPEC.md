# Phase 10 — UI Design Contract

> Visual and interaction contract for the Agentic Heartbeat Cap. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Adapted for GPU-rendered Rust desktop context (wgpu + glyphon + cosmic-text). No web technologies.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (GPU-rendered Rust app) |
| Preset | not applicable |
| Component library | wgpu QuadInstance + glyphon TextLabel |
| Icon library | Unicode glyphs (codepoints, no icon font) |
| Font | System SansSerif via cosmic-text FontSystem |

**Context:** All UI is rendered via `QuadInstance` for rectangles/backgrounds/borders and `TextLabel` for text. Colors come from the `Theme` struct (linear-light RGBA `[f32; 4]`). The active `ThemeDefinition` (Dracula default) provides all color tokens. No CSS, no HTML, no DOM.

---

## Spacing Scale

All values are in logical pixels (f32). Scale factor applied by glyphon at render time.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 2.0 px | Accent bar width, divider thickness |
| sm | 4.0 px | Vertical text centering offsets, tight internal padding |
| md | 8.0 px | Standard gap between elements, LEFT_PAD/RIGHT_PAD, dot gaps |
| lg | 16.0 px | Section padding, sidebar content indent, toast margin |
| xl | 24.0 px | Bar heights (stats bar, bottom bar, section headers) |
| 2xl | 28.0 px | Row height for list entries (matches ENTRY_HEIGHT_PX from sidebar) |
| 3xl | 32.0 px | Compact data rows (matches ROW_HEIGHT from agent_monitor) |

Exceptions:
- Touch/click target minimum: 28.0 px height (established by sidebar ENTRY_HEIGHT)
- Sidebar default width: 240.0 px (matches left sidebar SIDEBAR_DEFAULT_WIDTH)
- Sidebar minimum width: 160.0 px (matches SIDEBAR_MIN_WIDTH)
- Toast width: 280.0 px, toast height: 48.0 px (established pattern)

## Typography

All text rendered via `TextLabel` with `Family::SansSerif` and `Shaping::Advanced`. Line height is derived as `font_size * 1.3` by the `TextEngine::prepare` method.

| Role | Font Size | Weight | Line Height | Theme Color | Usage |
|------|-----------|--------|-------------|-------------|-------|
| Section Header | 12.0 px | SEMIBOLD (600) | 15.6 | `fg_secondary` | "JOBS", "RESULTS", "RECENT ALERTS" section labels |
| Body | 13.0 px | NORMAL (400) | 16.9 | `fg_primary` | Job names, result text, message content, values |
| Small | 11.0 px | NORMAL (400) | 14.3 | `fg_secondary` | Timestamps, labels ("Last run:"), stats bar labels, attribution |
| Empty State Title | 14.0 px | NORMAL (400) | 18.2 | `fg_primary` | Empty state heading text |

Source: These sizes match the established codebase patterns exactly. 12.0 for section headers (sidebar "FILES", agent monitor "RECENT ALERTS"), 13.0 for body (status bar values, agent names, toast messages), 11.0 for small labels (stats bar labels, detail text, alert timestamps), 14.0 for entry text (sidebar file entries, empty state).

Colors are theme-derived. All values reference `Theme` struct fields, which resolve to different hex values per active theme. The contract specifies field names, not hex codes, because the app supports 4+ themes.

| Role | Theme Field | Dracula Hex | Usage |
|------|-------------|-------------|-------|

| Severity: Critical | `error` | #FF5555 | [CRITICAL] severity tag dot, critical result border accent |
| Severity: Warning | `warning` | #FFB86C | [WARNING] severity tag dot, warning result border accent |

## Surface Specifications

### Surface 1: Right Sidebar (Job Browser)

**Layout:** Mirrors left sidebar architecture. Anchored to right edge of viewport, full height from stats bar to bottom bar. Default width 240.0 px, resizable with 4.0 px edge hit zone, min 160.0 px, max 40% of window width.

| Element | Position | Size | Details |
|---------|----------|------|---------|
| Sidebar background | Right edge, full viewport height | 240.0 x viewport_h | `theme.panel_background` |
| "HEARTBEATS" header | top + 16.0 px | 12.0 px SEMIBOLD | `fg_secondary`, same style as sidebar "FILES" |
| Active count badge | Right of header | 11.0 px | `divider_hover` accent, format: "[N running]" |
| Job list start | header_y + 15.6 + 8.0 | -- | Below header with 8.0 px gap |
| Job row | Full width | 28.0 px height | Matches ENTRY_HEIGHT_PX |
| Status dot | 8.0 px from left + indent | 8.0 x 8.0 px | Corner radius 4.0 (circle), severity-colored |
| Job name | After dot + 8.0 px gap | 13.0 px NORMAL | `fg_primary` (enabled) or `fg_secondary` (disabled) |
| Last run time | Right-aligned - 8.0 px | 11.0 px | `fg_secondary`, relative time ("2m ago") |
| Selected row | Full width | 28.0 px | `sidebar_selected_bg` + 2.0 px accent left bar |
| Hovered row | Full width | 28.0 px | `sidebar_hover_bg` |
| Section divider | Full width | 1.0 px | `border` color, between job list and actions |
| "Run Now" button | Below selected job | 28.0 px height | `divider_hover` text, appears when job is selected |
| Enable/Disable toggle | Inline right side of row | 11.0 px text | "ON" in `success`, "OFF" in `fg_secondary` |

**Interaction:**
- Click job row: select job, show "Run Now" button and detail area
- Double-click job row: open heartbeat output cap in focused panel split
- Right-click job row: context menu (Run Now, Enable/Disable, Edit)
- Scroll: mouse wheel scrolls job list with viewport culling
- Resize: drag left edge of sidebar (same pattern as left sidebar right edge)

**Inline Editor (expanded below selected job):**
When user selects "Edit" from context menu, sidebar expands an edit section below the selected job row:

| Field | Input Style | Size |
|-------|-------------|------|
| Prompt template | Multi-line text area (bg_secondary background) | Min 3 rows x sidebar width - 32.0 px |
| File patterns | Single-line text, comma-separated | 1 row |
| Schedule interval | Label + value ("every Nm") | 1 row |
| Watch paths | Single-line text, comma-separated | 1 row |
| Save button | "Save" text in `divider_hover` | 28.0 px row |
| Cancel button | "Cancel" text in `fg_secondary` | 28.0 px row |

Edit section background: `bg_secondary` with 4.0 px corner radius, 8.0 px internal padding.

### Surface 2: Heartbeat Output Cap (Grid Panel)

**Layout:** Standard grid panel with title bar. One cap per job. Shows that job's latest result and scrollable run history.

| Element | Position | Size | Details |
|---------|----------|------|---------|
| Panel title bar | Top of panel bounds | 28.0 px (PANEL_TITLE_HEIGHT) | Standard panel chrome, title = job name |
| Content background | Below title bar | panel_w x remaining_h | `panel_background` |
| Latest result header | content_top + 8.0 px | 12.0 px SEMIBOLD | "LATEST RESULT" in `fg_secondary` |
| Severity badge | Right of header | 11.0 px text | Colored by severity: "[CRITICAL]" / "[WARNING]" / "[INFO]" |
| Result timestamp | Below header | 11.0 px | `fg_secondary`, absolute time + relative ("2m ago") |
| Result body text | Below timestamp + 8.0 px | 13.0 px, wrapping | `fg_primary`, full result text with line wrapping |
| Severity accent bar | Left edge of result area | 2.0 px x result_height | Severity-colored (error/warning/success) |
| History divider | Below result body + 16.0 px | 1.0 px full width | `border` |
| "HISTORY" header | Below divider + 8.0 px | 12.0 px SEMIBOLD | `fg_secondary` |
| History rows | Below history header | 24.0 px each | Compact: timestamp (11px) + severity dot (8x8) + first line (13px, truncated) |
| Alternating rows | Every other row | 24.0 px | `bg_secondary` at 0.3 alpha (matches agent monitor pattern) |

**Interaction:**
- Scroll: mouse wheel scrolls result body and history independently (two scroll zones)
- Click history row: replaces latest result view with the clicked historical result
- Standard panel interactions: close, split, resize, fullscreen (inherited from grid)

**States:**

| State | Display |
|-------|---------|
| Running (spinner) | Centered "Running..." text (14.0 px, `fg_secondary`) with pulsing dot (8x8, `divider_hover`, alpha oscillation 0.3-1.0) |
| Completed result | Full result display as specified above |
| No results yet | Empty state (see Copywriting section) |
| Error (LLM failure) | Error message with `error` accent bar, retry guidance text |
| Job disabled | Greyed content, "Job disabled" label centered |

### Surface 3: Top Bar Heartbeat Indicator

**Layout:** New slot in existing `StatsBar` system. Occupies one of the reserved invisible slots (index 2 or 3).

| Element | Position | Size | Details |
|---------|----------|------|---------|
| Pulsing dot | Left side of slot | 6.0 x 6.0 px | `divider_hover` (accent), corner_radius 3.0 (circle) |
| Running count | Right of dot + 4.0 px | 11.0 px label + 13.0 px value | Label: `fg_secondary`, Value: `fg_primary` |

**Pulsing animation:** Dot alpha oscillates between 0.4 and 1.0 using `sin(elapsed_secs * 3.0)` (1.5 Hz pulse, same vibe as a heartbeat). Only pulses when at least one job is actively running. Solid alpha 1.0 when idle with completed results. Hidden (slot invisible) when no heartbeat jobs are configured.

**Slot format:** Label: "HB" (abbreviated for 24px bar space), Value: "N running" or "idle"

**Interaction:** Click anywhere in the stats slot opens/focuses the right sidebar. Uses existing `StatsSlot` architecture -- the slot is made visible when heartbeat jobs exist.

## Interaction Patterns

### Right Sidebar Toggle
- **Shortcut:** Cmd+Shift+B (mirrors Cmd+B for left sidebar, with Shift modifier for right)
- **Add to InputAction enum:** `ToggleRightSidebar`
- **Animation:** None (instant show/hide, matches left sidebar)

### Job Selection Flow
1. User clicks job row in right sidebar -- row highlights with `sidebar_selected_bg` + 2px accent bar
2. "Run Now" text button appears below selected job
3. Double-click opens heartbeat output cap as new split in focused panel
4. Right-click shows context menu with: Run Now, Enable/Disable, Edit Job, Open in Panel

### Heartbeat Toast Integration
- Toast type: `ToastType::Intervention` for CRITICAL, `ToastType::Info` for WARNING
- INFO results do not trigger toasts (ambient only, visible in cap)
- Toast `source_panel`: None (heartbeat is not tied to a specific panel)
- Toast `pattern_id`: `heartbeat_{job_name}` (enables per-job rate limiting)
- Toast `attribution`: model name from provider config
- Toast duration: `INTERVENTION_TOAST_DURATION` (8 seconds) for CRITICAL, `INFO_TOAST_DURATION` (3 seconds) for WARNING
- Clicking toast opens/focuses the right sidebar and selects the relevant job

### Scroll Behavior
- Right sidebar: single scroll zone for job list, viewport culling (matches left sidebar)
- Heartbeat cap: two scroll zones separated by history divider. Top zone scrolls result body, bottom zone scrolls history list. Determined by mouse Y position relative to divider.

## Animation Specifications

| Animation | Property | Range | Frequency | Trigger |
|-----------|----------|-------|-----------|---------|
| Stats bar heartbeat dot | Alpha | 0.4 - 1.0 | sin(t * 3.0), ~1.5 Hz | At least one job actively running |
| Running spinner dot (cap) | Alpha | 0.3 - 1.0 | sin(t * 4.0), ~2 Hz | Job is executing |

Implementation: Both animations use `Instant::now().elapsed().as_secs_f32()` in the render loop to compute alpha. Applied to `QuadInstance.color[3]` (alpha channel). No spring physics or easing -- simple sinusoidal oscillation.
