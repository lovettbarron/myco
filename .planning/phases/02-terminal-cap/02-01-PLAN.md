1. **Create `src/terminal/renderer.rs` -- GPU character grid renderer:**

   Define `pub struct TerminalRenderer` with:
   - `buffers: Vec<glyphon::Buffer>` -- one per visible row, cached between frames
   - `font_size: f32` -- current font size (default 14.0)
   - `cell_width: f32` -- computed from font metrics
   - `cell_height: f32` -- computed from font metrics (font_size * 1.3 line height)
   - `palette: crate::terminal::colors::AnsiPalette`

   Implement:
   - `pub fn new() -> Self` -- initialize with AnsiPalette::default(), font_size 14.0
   - `pub fn compute_cell_dimensions(font_system: &mut FontSystem, font_size: f32) -> (f32, f32)` -- create a temporary Buffer with Metrics::new(font_size, font_size * 1.3), set_text with "M" in Family::Monospace, shape it, read the advance width from the first layout run. Return (advance_width, line_height). If advance width cannot be determined, fallback to (font_size * 0.6, font_size * 1.3).
   - `pub fn prepare_terminal_text(&mut self, font_system: &mut FontSystem, term: &Term<MycoEventListener>, viewport_x: f32, viewport_y: f32, viewport_w: f32, viewport_h: f32) -> Vec<glyphon::TextArea<'_>>` -- Lock term briefly, read RenderableContent, build per-row Buffers with set_rich_text:
     - For each visible row (from content.display_iter()):
       - Group cells into spans by foreground color attribute
       - Skip cells with Flags::WIDE_CHAR_SPACER (per Pitfall 3)
       - For WIDE_CHAR cells, include the character (cosmic-text handles width)
       - Build spans as Vec<(&str, Attrs)> where Attrs has Family::Monospace and color from resolve_fg()
       - Call buffer.set_rich_text(font_system, spans, default_attrs, Shaping::Advanced, None)
       - Call buffer.shape_until_scroll(font_system, false)
     - Return TextArea items with proper left/top positioning based on row index * cell_height + viewport_y offset
   - `pub fn build_terminal_quads(&self, term: &Term<MycoEventListener>, viewport_x: f32, viewport_y: f32, viewport_w: f32, viewport_h: f32, panel_bg: [f32; 4]) -> Vec<QuadInstance>` -- produces quads for:
     - Cell backgrounds: only render quads where cell.bg differs from the panel background color (per Pitfall 5). Convert cell bg from [u8;3] to [f32;4].
     - Cursor quad (per D-07 default solid block, TERM-08 style switching):
       - Read cursor from RenderableContent (cursor position, shape)
       - CursorShape::Block: solid filled quad at cursor position, sized cell_width x cell_height, foreground color
       - CursorShape::Beam: thin quad (2px wide) at cursor x, full cell_height
       - CursorShape::Underline: thin quad (2px tall) at cursor bottom

     - For cursor blink: accept a `cursor_visible: bool` parameter

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PTY output -> VTE parser | Untrusted data from shell subprocess crosses into terminal state |
| User keyboard -> PTY input | User input written directly to PTY (pass-through, no injection risk) |
| Environment inheritance | Parent process env passed to child shell (intentional per D-04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | PTY output parsing | mitigate | alacritty_terminal's battle-tested VTE parser handles all escape sequence parsing. Never parse escape sequences manually. |
| T-02-02 | Tampering | OSC 52 clipboard write | mitigate | Do not implement OSC 52 clipboard write in this phase. Programs cannot write to clipboard via escape sequences. |
| T-02-03 | Information Disclosure | Environment variables | accept | D-04 explicitly requires full env inheritance. The shell runs at the same privilege level as the user. No elevation. |
| T-02-04 | Denial of Service | Unbounded PTY output | mitigate | alacritty_terminal's scrollback buffer is capped at 50K lines (D-12). FairMutex prevents render thread starvation. |
| T-02-05 | Elevation of Privilege | Shell command execution | accept | Terminal runs user's own shell at user's own privilege level. No additional attack surface beyond what the user already has. |
</threat_model>
