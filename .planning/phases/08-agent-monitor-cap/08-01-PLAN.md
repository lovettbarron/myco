5. The process tree walk algorithm:
   ```rust
   fn is_descendant_of(system: &System, pid: Pid, ancestor_pid: u32, max_depth: u8) -> bool {
       let mut current = Some(pid);
       let mut depth = 0;
       while let Some(p) = current {
           if depth > max_depth { return false; }
           if let Some(proc_info) = system.process(p) {
               if let Some(parent) = proc_info.parent() {
                   if parent.as_u32() == ancestor_pid { return true; }
                   current = Some(parent);
               } else {
                   return false;
               }
           } else {
               return false;
           }
           depth += 1;
       }
       false
   }
   ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system -> app | agents.json loaded from ~/.myco/ could contain malicious content |
| Process table -> app | sysinfo data is OS-provided, trusted, but agent name matching is string comparison against user config |
| Terminal text -> token parser | Terminal output is untrusted user-controlled content fed to substring parser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | DoS | AgentConfig::load() | mitigate | File size limit 1MB, max 100 entries, max 200 chars per process name string |
| T-08-02 | Tampering | agents.json path | mitigate | Fixed path only (~/.myco/agents.json), never user-supplied. No path traversal possible. |
| T-08-03 | DoS | Agent discovery | mitigate | Cap max displayed agents at 50. Process tree walk limited to depth 5. |
| T-08-04 | DoS | Token parsing | accept | Substring search on terminal text is O(n) with small constant. No regex. Terminal text already capped by scrollback buffer size. |
| T-08-05 | DoS | Alert history | mitigate | Capped at 50 entries maximum in memory. Oldest entries dropped. |
</threat_model>
