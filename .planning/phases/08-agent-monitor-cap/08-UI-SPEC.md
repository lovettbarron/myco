# Phase 8 — UI Design Contract

> Visual and interaction contract for the Agent Monitor Cap phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (GPU-rendered via wgpu + QuadInstance + glyphon) |
| Preset | not applicable |
| Component library | none (custom GPU primitives: QuadInstance, TextLabel, glyphon Buffers) |
| Icon library | none (Unicode glyphs for indicators: status dots via QuadInstance circles, chevrons via text) |
| Font | System default via cosmic-text FontSystem (SansSerif family, same as sidebar/picker) |

---

## Spacing Scale

Declared values (must be multiples of 4). Consistent with Phase 6 UI-SPEC and existing codebase constants.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Status dot diameter (radius 2.0 for QuadInstance), inline gaps between stats values, chevron-to-text gap |
| sm | 8px | Row internal horizontal padding, gap between columns, gap between alert log entries, content inset from panel edges |
| md | 16px | Panel header padding, section divider vertical spacing, left indent for detail section |
| lg | 24px | Panel header height (reuse STATS_BAR_HEIGHT pattern), section header height |
| xl | 32px | Compact row height (accommodates status dot + name + stats in one line with vertical padding) |
| 2xl | 48px | Expanded detail section height per detail row |

Exceptions:
- Status dot diameter: 8px (consistent with Phase 6 resource dot)
- Status dot hit area: 16x16px (expanded for click targeting, same as Phase 6)
- Sparkline widget: 64px wide x 16px tall (compact inline chart)
- Divider line between agent list and alert log: 1px height, full width
- Badge pill in panel header: height 16px, horizontal padding 6px (6px is 1.5x xs, acceptable for pill aesthetics)
- Chevron glyph touch target: 24x32px (full row height, left 24px)

## Typography

All text rendered via glyphon + cosmic-text. Font sizes in CSS px (converted to cosmic-text Metrics internally). Matches the established 4-size scale from Phase 6 and existing codebase.

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 13px | 400 (regular) | 1.4 (18.2px) | Agent name in compact row, stats values (CPU%, RAM, tokens), alert log messages, panel header title |
| Label | 11px | 400 (regular) | 1.3 (14.3px) | Running time, token breakdown labels in detail view, alert timestamps, "Recent Alerts" section header, empty state body text, badge count text |
| Heading | 14px | 600 (semibold) | 1.2 (16.8px) | Agent name in expanded row, detail section labels ("Tokens", "CPU History", "Alerts"), empty state heading |
| Small | 12px | 400 (regular) | 1.3 (15.6px) | Column headers ("STATUS", "AGENT", "TIME", etc.), stat unit suffixes ("MB", "tk"), section divider label |

Two weights only: 400 (regular) for all body/label/small text, 600 (semibold) for headings and section headers. Consistent with sidebar renderer pattern (Weight::NORMAL and Weight::SEMIBOLD).

Colors are theme-derived. All colors reference existing `Theme` struct fields so they adapt across Dracula, Solarized Dark, Solarized Light, and Obsidian themes. No new Theme fields required beyond those added in Phase 6.

| Role | Theme Field | Dracula Hex | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | `background` / `panel_background` | #282A36 / #2C2E3B | Panel body background, empty state background |
| Secondary (30%) | `bg_secondary` | #2C2E3B | Compact row hover background, expanded detail section background, alert log entry background, badge pill background |
| Accent (10%) | `divider_hover` (accent) | #BD93F9 | Selected row left accent bar (2px), clickable agent name on hover, "Focus Terminal" action in context menu, badge pill text color, chevron glyph color on hover |
| Destructive | `error` | #FF5555 | Kill action in context menu (text color), critical status dot |

## Interaction Contracts

### Open Agent Monitor Panel

1. User presses Cmd+Shift+A (D-15) or selects "Agent Monitor" from panel type picker
2. New panel created with PanelType::AgentMonitor
3. Panel immediately begins receiving ResourceUpdate and InterventionAlert events from existing monitor thread
4. Agent discovery runs on next poll cycle (within 2 seconds)
5. Panel renders empty state if no agents detected

### Click Agent Row -> Focus Terminal

1. User single-clicks anywhere on a compact agent row (excluding the chevron hit zone, 24px from left edge)
2. Source terminal PanelId looked up from AgentSession
3. Grid scrolls to make source panel visible if needed
4. Source panel receives keyboard focus
5. If source panel was offscreen, grid layout adjusts to show it (existing grid focus behavior)

### Click Chevron -> Expand/Collapse Detail

1. User clicks within the 24x32px chevron hit zone (left edge of row)
2. Chevron glyph toggles: U+25B8 -> U+25BE (or reverse)
3. Detail section slides into view below the compact row (no animation, instant expand for v1)
4. Other expanded rows remain expanded (multiple rows can be expanded simultaneously)
5. Panel content area re-layouts, scroll position adjusts to keep the expanded row visible

### Right-Click Agent Row -> Context Menu

1. User right-clicks (or Ctrl+Click) anywhere on an agent row
2. Native NSMenu appears at click coordinates
3. Menu items depend on agent state (running/frozen/idle per menu items table above)
4. User selects "Focus Terminal" -> same as single-click focus behavior
5. User selects "Freeze Process" -> SIGSTOP sent via existing freeze_process_group(), status dot updates to frozen color
6. User selects "Unfreeze Process" -> SIGCONT sent via existing unfreeze_process_group(), status dot updates
7. User selects "Kill Process" -> native NSAlert confirmation dialog. If confirmed, SIGKILL sent to process group. Agent removed from list on next poll cycle.
8. User selects "Copy Stats" -> formatted stats string copied to clipboard via NSPasteboard

### Alert Log Click -> Focus Terminal

1. User clicks on an alert log entry
2. Source panel from the original InterventionAlert looked up
3. Same focus behavior as clicking an agent row

### Scroll Behavior

1. Mouse wheel event received while cursor is within agent monitor panel
2. Determine which region cursor is in: agent list (top 60%) or alert log (bottom 40%)
3. Apply scroll delta to the correct region's scroll_offset
4. Clamp scroll_offset to [0, max_content_height - visible_height]
5. Pixel accumulator pattern: 20.0px per scroll line (matches sidebar)

### Keyboard Shortcut (Cmd+Shift+A)

1. Shortcut registered in ShortcutRegistry as InputAction::OpenAgentMonitor
2. If no AgentMonitor panel exists in the grid -> create one in the focused column (split)
3. If an AgentMonitor panel already exists -> focus it
4. Only one AgentMonitor panel per workspace (singleton pattern)

## Rendering Implementation Notes

### Recommended Module Structure

```
src/agent_monitor/
    mod.rs          -- AgentMonitorState, AgentSession, discovery logic
    renderer.rs     -- build_quads(), build_labels() following picker/sidebar pattern
```

### Rendering Pattern

Follow the established pattern from `picker/renderer.rs` and `sidebar/renderer.rs`:

1. `build_quads(state, viewport, theme) -> Vec<QuadInstance>` for backgrounds, dots, sparkline bars, dividers
2. `build_labels(state, viewport, theme) -> Vec<TextLabel>` for all text content
3. Both functions called from `app.rs` rendering dispatch, same as other panel types
4. Hit testing via computed rects stored in state, same pattern as `SidebarState::handle_click()`

### Sparkline Rendering

The CPU sparkline is rendered as 30 adjacent 2px-wide QuadInstance bars (total 60px, within 64px allocation):

- Each bar height = (cpu_sample / 100.0) * 16.0, clamped to [1.0, 16.0]
- Each bar color = `dot_color(cpu_sample, theme)` (green/yellow/red per threshold)
- Bars positioned left-to-right, oldest sample on left
- 1px gap between bars (2px bar + 0px gap = 2px per sample, 30 * 2 = 60px)
