**src/grid/operations.rs** -- Per PATTERNS.md operations section and decisions D-08 through D-11:

`pub fn split_panel(grid: &mut GridLayout, panel_id: PanelId, direction: SplitDirection) -> Option<PanelId>`
- Per D-08: split an existing panel, creating a new one
- Find the panel's NodeId in grid
- Get current root style grid_template_columns/rows
- For horizontal split: add a new `fr(1.0)` column track, create new leaf node, add as child of root, assign grid_column placement to both old and new panels so they sit in correct columns
- For vertical split: add a new `fr(1.0)` row track, create new leaf, assign grid_row placement
- Return the new PanelId
- `pub enum SplitDirection { Horizontal, Vertical }`

`pub fn close_panel(grid: &mut GridLayout, panel_id: PanelId) -> bool`
- Per D-09: neighbor with most shared edge absorbs space
- Find panel's NodeId, determine which track (column or row) it occupies
- Find the neighbor that shares the most edge: look at adjacent panels in the same row/column, pick the one with the largest shared dimension
- Remove the panel's NodeId from the taffy tree
- Remove the corresponding track from grid_template_columns/rows
- Update remaining panel placements to close the gap
- Return true if closed successfully, false if it's the last panel (can't close)

`pub fn swap_panels(grid: &mut GridLayout, panel_a: PanelId, panel_b: PanelId)`
- Per D-10: swap content/identity, preserve grid structure
- Find both NodeIds
- Swap the PanelId values in grid.panels (the NodeIds stay in their grid positions, the PanelIds move)

`pub fn toggle_fullscreen(grid: &mut GridLayout, panel_id: PanelId) -> bool`
- Per D-11: in-window fullscreen, save state, restore on toggle
- If no fullscreen active: save current columns, rows, and panels. Set grid to single column/row with only the target panel visible. Set fullscreen_state.
- If fullscreen active (same panel): restore saved columns, rows, panels. Clear fullscreen_state.
- If fullscreen active (different panel): just switch which panel is shown.
- Return true if now fullscreened, false if restored.

- After grid.compute(), iterate the track boundaries to find where dividers sit

`pub fn hit_test_divider(dividers: &DividerSet, cursor_x: f32, cursor_y: f32) -> Option<(usize, Orientation)>`

`pub fn apply_divider_drag(grid: &mut GridLayout, orientation: Orientation, track_index: usize, delta_pixels: f32, total_track_size: f32)`
- Per D-05: redistribute ALL fr() values in the affected dimension proportionally
- Convert delta_pixels to a fraction of total_track_size
- Split the fr values at track_index: left-side tracks and right-side tracks
- Scale left-side tracks by (left_total + delta_fraction) / left_total
- Scale right-side tracks by (right_total - delta_fraction) / right_total
- Per D-06: clamp so no individual track's fr value would result in a pixel size below PANEL_MIN_SIZE
- Per D-07: this is called on every mouse move (live resize)

```rust
use winit::event::{ElementState, KeyEvent};
use winit::keyboard::{Key, NamedKey, ModifiersState};
use super::InputAction;
use crate::grid::PanelId;

pub fn handle_key_event(
    event: &KeyEvent,
    modifiers: &ModifiersState,
    focused_panel: Option<PanelId>,
) -> Option<InputAction> {
    if event.state != ElementState::Pressed { return None; }
    let panel_id = focused_panel?;
    match &event.logical_key {
        Key::Named(NamedKey::Escape) => Some(InputAction::PanelToggleFullscreen { panel_id }),
        Key::Character(c) if modifiers.super_key() => {
            match c.as_str() {
                "d" => Some(InputAction::PanelSplitHorizontal { panel_id }),
                "D" => Some(InputAction::PanelSplitVertical { panel_id }),  // Cmd+Shift+D
                "w" => Some(InputAction::PanelClose { panel_id }),
                _ => None,
            }
        }
        _ => None,
    }
}
```
Shortcuts: Cmd+D = split horizontal, Cmd+Shift+D = split vertical, Cmd+W = close panel, Escape = toggle fullscreen. Right-click = directional split (see mouse.rs above).
