Create the Rust project scaffold with the GPU rendering pipeline core. Follow RESEARCH.md patterns exactly. File-by-file specification:

**Cargo.toml** -- Per RESEARCH.md lines 784-816:
```toml
[package]
name = "myco"
version = "0.1.0"
edition = "2021"

[dependencies]
wgpu = "29.0.3"
glyphon = "0.11.0"
pollster = "0.4"
winit = "0.30.13"
taffy = { version = "0.10.1", features = ["grid"] }
serde = { version = "1", features = ["derive"] }
serde_json = "1.0.149"
tracing = "0.1.44"
tracing-subscriber = { version = "0.3", features = ["env-filter"] }
bytemuck = { version = "1", features = ["derive"] }

[target.'cfg(target_os = "macos")'.dependencies]
objc2 = "0.6.4"
objc2-app-kit = "0.3.2"
objc2-foundation = "0.3.2"
```
Note: bytemuck is needed for QuadInstance Pod/Zeroable derives (used by wgpu vertex buffers).

**src/main.rs** -- Per PATTERNS.md src/main.rs section:
- Initialize tracing with EnvFilter (use `use tracing::info;` to confirm tracing is wired)
- Create EventLoop
- Create App::default()
- Call event_loop.run_app(&mut app)
- Minimal -- all state lives in App

**src/app.rs** -- Per PATTERNS.md src/app.rs section:
- `App` struct with fields: `window: Option<Arc<Window>>`, `renderer: Option<Renderer>`, `theme: Theme`
- Note: grid and panels fields will be added by Plan 01-02. For now, App just renders a solid background.
- Implement `ApplicationHandler` for `App`:
  - `resumed()`: create window via `window::create_window()`, init GPU via `GpuState::new()`, init `Renderer`, call `platform::macos::setup_custom_title_bar()` (per D-14). Also re-apply traffic light positioning after initial setup to handle layout settling (per Open Question 4 resolution).
  - `window_event()`: handle CloseRequested (exit), Resized (guard width>0 && height>0, then resize GPU surface), RedrawRequested (render frame with solid background color from theme)
  - `about_to_wait()`: request_redraw
- On RedrawRequested, render a solid theme.background color via the renderer's clear color. No quads yet -- that comes in Plan 01-02.

**src/window.rs** -- Per PATTERNS.md src/window.rs section:
- `pub fn create_window(event_loop: &ActiveEventLoop) -> Arc<Window>` function
- WindowAttributes with title "Myco", surface_size LogicalSize(1280, 800), min_surface_size LogicalSize(640, 480)
- On macOS: `with_titlebar_transparent(true)`, `with_fullsize_content_view(true)`, `with_title_hidden(true)` (per D-14). Do NOT use `with_decorations(false)` -- that removes traffic lights (Pitfall 3)
- Center on screen at ~80% size per D-13: get current_monitor().size(), calculate 80% dimensions, call request_surface_size

**src/renderer/mod.rs** -- Per PATTERNS.md:
- `pub mod gpu_state;`
- `Renderer` struct owning `GpuState`
- `Renderer::new(window: Arc<Window>) -> Self`: init GpuState via `pollster::block_on(GpuState::new(window))`
- `Renderer::resize(&mut self, width: u32, height: u32)`: delegate to gpu_state.resize()
- `Renderer::render(&mut self, clear_color: [f32; 4]) -> Result<(), wgpu::SurfaceError>`: get current texture, create encoder, begin render pass with clear color, submit, present. QuadRenderer and TextEngine integration will be added by Plan 01-02.
- Expose `pub fn device(&self)`, `pub fn queue(&self)`, `pub fn format(&self)` accessors for downstream renderers to use.

**src/renderer/gpu_state.rs** -- Per PATTERNS.md gpu_state section exactly:
- `GpuState` struct with surface, device, queue, config, window, format (store the TextureFormat for later use)
- `async fn new(window: Arc<Window>)`: wgpu::Instance with PRIMARY backends, create_surface, request_adapter with compatible_surface, request_device with default limits/features, configure surface with sRGB format preference + AutoVsync present mode
- `fn resize(&mut self, width: u32, height: u32)`: guard width>0 && height>0 before reconfigure

**src/shaders/quad.wgsl** -- Copy RESEARCH.md Pattern 4 WGSL shader verbatim (lines 521-579):
- QuadInstance struct with @location(0) through @location(3) for position, size, color, corner_radius
- Uniforms with viewport_size
- QUAD_VERTICES private array with 6 vertices forming two triangles
- vs_main: transform pixel coordinates to clip space
- fs_main: return solid color

**src/theme.rs** -- Per PATTERNS.md theme section:
- `Theme` struct with fields: background, panel_background, title_bar_text, divider, divider_hover, panel_label_text -- all [f32; 4]
- `Theme::dark()` returning the dark theme colors from PATTERNS.md:
  - background: [0.1, 0.1, 0.12, 1.0]
  - panel_background: [0.14, 0.14, 0.16, 1.0]
  - title_bar_text: [0.78, 0.78, 0.80, 1.0]
  - divider: [0.2, 0.2, 0.22, 1.0]
  - divider_hover: [0.4, 0.4, 0.45, 1.0]
  - panel_label_text: [0.5, 0.5, 0.52, 1.0]
- `impl Default for Theme { fn default() -> Self { Self::dark() } }`

Critical anti-patterns to avoid:
- Do NOT use `with_decorations(false)` -- removes traffic lights (Pitfall 3)
- Do NOT use `can_create_surfaces()` -- that is winit 0.31 beta, not 0.30.13 stable (Pitfall from RESEARCH State of the Art)
- Do NOT forget `features = ["grid"]` on taffy (Pitfall 5)
- Do NOT pass zero dimensions to wgpu (Pitfall 2) -- guard all resize/configure calls
- Use `Arc<Window>` for surface lifetime (Pitfall 4)
- Use `pollster::block_on()` only for one-time wgpu async init, never in render loop
