**src/app.rs** -- Per PATTERNS.md src/app.rs section:
- `App` struct with fields: `window: Option<Arc<Window>>`, `renderer: Option<Renderer>`, `grid: Option<GridLayout>`, `panels: Vec<Panel>`, `theme: Theme`
- Implement `ApplicationHandler` for `App`:
  - `resumed()`: create window via `window::create_window()`, init GPU via `GpuState::new()`, init `Renderer`, init `GridLayout::new_single_panel()`, create one `Panel { id: PanelId(0), panel_type: PanelType::Placeholder, title: "Placeholder".into() }`, call `platform::macos::setup_custom_title_bar()` (per D-14)
  - `window_event()`: handle CloseRequested (exit), Resized (guard width>0 && height>0, then resize GPU surface + recompute grid layout), RedrawRequested (build draw data from grid layout + panels + theme, pass to renderer, render frame)
  - `about_to_wait()`: request_redraw
- On RedrawRequested, the render sequence is:
  1. Get window physical size
  2. Call `grid.compute(width as f32, height_minus_titlebar as f32)` -- reserve top 38px for custom title bar area
  3. For each panel, get rect from `grid.get_panel_rect(node)`, offset y by title bar height (38px)
  4. Build `Vec<QuadInstance>` for: title bar background quad (full width, 38px tall, theme.background color), panel background quads (theme.panel_background color), panel title bar quads (thin area at top of each panel for the cap label per D-01/D-02)
  5. Call `renderer.render(quads, window_width, window_height)`

**src/renderer/quad_renderer.rs** -- Per PATTERNS.md quad_renderer section + RESEARCH Pattern 4:
- `QuadInstance` struct: `#[repr(C)] #[derive(Copy, Clone, Debug, bytemuck::Pod, bytemuck::Zeroable)]` with position: [f32; 2], size: [f32; 2], color: [f32; 4], corner_radius: f32, plus padding: f32 for 16-byte alignment
- `Uniforms` struct: `#[repr(C)] #[derive(Copy, Clone, bytemuck::Pod, bytemuck::Zeroable)]` with viewport_size: [f32; 2]
- `QuadRenderer` struct with pipeline, uniform_buffer, uniform_bind_group, instance_buffer, instance_count
- `QuadRenderer::new(device, format)`: create shader module from include_str!("../shaders/quad.wgsl"), create uniform buffer, create bind group layout + bind group for uniforms, create render pipeline with vertex state (no vertex buffers, instances via @location attributes on the instance buffer), instance buffer layout matching QuadInstance fields, fragment state targeting the surface format
- Instance buffer layout: step_mode = Instance, attributes at locations 0-4 mapping to position, size, color, corner_radius
- `prepare(&mut self, device, queue, quads: &[QuadInstance], viewport_width, viewport_height)`: write uniforms to queue, recreate or write instance buffer with quad data, store instance_count
- `render<'pass>(&'pass self, pass: &mut RenderPass<'pass>)`: set pipeline, set bind group 0, set vertex buffer 0 (instance buffer), draw(0..6, 0..instance_count)
