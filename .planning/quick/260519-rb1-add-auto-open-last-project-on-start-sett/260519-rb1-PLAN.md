Purpose: Power users who work on one project most of the time skip the picker entirely.
Output: Modified `src/config/global.rs`, `src/settings.rs`, `src/app.rs`.

Replace the block that checks `if cli_project_dir.is_none()` with logic that also checks the new preference. The revised flow:

```rust
if cli_project_dir.is_none() {
    // Check if we should auto-open the last project
    let global_prefs = crate::config::global::load_global_preferences();
    if global_prefs.open_last_project {
        // Find the most recently opened project that still exists on disk
        let mut candidates: Vec<_> = self.project_registry.projects.iter()
            .filter(|p| p.exists() && p.last_opened.is_some())
            .collect();
        candidates.sort_by(|a, b| b.last_opened.cmp(&a.last_opened)); // descending by timestamp

        if let Some(last_project) = candidates.first() {
            let project_path = last_project.path.clone();
            info!("Auto-opening last project: {:?}", project_path);
            // Fall through to workspace init with this path
            // (set cli_project_dir equivalent so the rest of init proceeds)
            // We need to set project_dir and NOT enter picker mode
        }
    }
    // ... if no auto-open, show picker as before
}
```

The cleanest approach: refactor the existing if/else so that:
1. First, determine `cli_project_dir` from args (existing code, unchanged)
2. If `cli_project_dir.is_none()`, check `open_last_project` pref and look up last project
3. If a last project is found and exists, treat it the same as a CLI-provided path (assign to a `resolved_project` variable)
4. If no last project or setting is off, show picker (existing code)
5. Otherwise proceed with workspace init (existing code)

Concretely, after line 4030 (`let cli_project_dir = ...`), add:

```rust
// Check auto-open-last-project preference when no CLI arg
let auto_open_dir = if cli_project_dir.is_none() {
    let global_prefs = crate::config::global::load_global_preferences();
    if global_prefs.open_last_project {
        let mut candidates: Vec<_> = self.project_registry.projects.iter()
            .filter(|p| p.exists() && p.last_opened.is_some())
            .collect();
        candidates.sort_by(|a, b| b.last_opened.cmp(&a.last_opened));
        candidates.first().map(|p| p.path.clone())
    } else {
        None
    }
} else {
    None
};
```

Then change the picker check from `if cli_project_dir.is_none()` to `if cli_project_dir.is_none() && auto_open_dir.is_none()`.

And in the workspace init section (line 4072+), change the project_dir computation to also consider `auto_open_dir`:

```rust
let project_dir = if let Some(ref cli_path) = cli_project_dir {
    // existing CLI path logic
} else if let Some(ref auto_path) = auto_open_dir {
    auto_path.clone()
} else {
    std::env::current_dir().unwrap_or_else(|_| std::path::PathBuf::from("/"))
};
```

This ensures auto-opened projects go through the same workspace init path (register, load config, apply theme, etc.) as CLI-opened projects.

**Important:** The global_prefs load for theme fallback later in the init (around line 4049 and 4092) should not conflict since those are separate loads. The auto_open_dir computation loads prefs once, early, only when no CLI arg.
